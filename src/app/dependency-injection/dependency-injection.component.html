Dependency Injection-- Detailed Implementation

We use the @Inject parameter decorator to instruct Angular we want to resolve a token and inject a dependency into a constructor.

@injectable is not needed if all dependent has any decorators like @component or @decorators, otherwise need to explicitly tell angular to inject this service in dependent constructor. 

  constructor(@Inject(OtherService) otherSerivce:OtherService)

Declaring providers in ngModule and in component as provider and viewprovider

ngModule.providers:All components share same instance

vs

component.providers:Component and its children share same instance

vs

component.viewProviders: When ng-content tags used for child component, Only component have the instance
